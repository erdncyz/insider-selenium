Project Overview

This project is a comprehensive Selenium WebDriver test automation solution developed to test QA (Quality Assurance) positions on the Insider company's career website. The project follows modern test automation best practices and implements the Page Object Model (POM) design pattern for maintainable and scalable test code.

Key Features

  • Page Object Model: Clean separation of test logic and page interactions
  • Modern Selenium: Uses Selenium WebDriver 4.15.0 with latest features
  • Automatic Driver Management: WebDriverManager handles ChromeDriver automatically
  • Optimized Locators: XPath and CSS selectors for reliable element identification
  • Comprehensive Assertions: Multiple validation points throughout the test flow
  • Clean Code: Readable, maintainable, and well-documented codebase

Test Requirements

Original Test Requirements

  1. Visit https://useinsider.com/ and check if Insider home page is opened
  2. Select "Company" menu in the navigation bar, select "Careers" and check Career page, its Locations, Teams, and Life at Insider blocks are open
  3. Go to https://useinsider.com/careers/quality-assurance/, click "See all QA jobs", filter jobs by Location: "Istanbul, Turkey" and Department: "Quality Assurance", check the presence of the jobs list
  4. Check that all jobs' Position contains "Quality Assurance", Department contains "Quality Assurance", and Location contains "Istanbul, Turkey"
  5. Click "View Role" button and check that this action redirects to the Lever Application form page

Technical Requirements

  • Programming Language: Java with Selenium WebDriver (preferred)
  • Framework Restrictions: No BDD frameworks allowed (Cucumber, Quantum, Codeception, etc.)
  • Design Pattern: Must fully meet Page Object Model (POM) requirements
  • Element Locators: Optimized XPath and CSS selectors for buttons, dropdowns, and other elements
  • Assertions: Must use assertions in test cases for validation
  • Code Quality: Clean, readable, and maintainable code

Test Scenario Implementation

Current Test Flow

  1. Homepage Verification: Visit https://useinsider.com/ and verify the Insider home page is opened
  2. Navigation to Careers: Select "Company" menu → "Careers" and verify the Career page with Locations, Teams, and Life at Insider blocks
  3. QA Department Access: Navigate to QA department page and click "See all teams"
  4. QA Department Selection: Select "Quality Assurance" department
  5. QA Jobs Selection: Click "See all QA jobs" button
  6. Position Filtering: Filter positions by location and department
  7. Lever Page Verification: Verify Lever application form page loads

Technical Architecture

Architectural Approach

The project uses the Page Object Model (POM) design pattern, which provides:

  • Maintainability: Easy code changes and updates
  • Reusability: Test codes can be reused across different scenarios
  • Ease of Maintenance: Web elements managed from single locations
  • Test Stability: Reduced impact of UI changes

Layered Architecture

┌─────────────────────────────────────┐ │ Test Layer │ ← Test Scenarios (VerifyQAJobs) ├─────────────────────────────────────┤ │ Page Object Layer │ ← Page Objects (HomePage, CareersPage, etc.) ├─────────────────────────────────────┤ │ Base Test Layer │ ← Common Setup (BaseTest) ├─────────────────────────────────────┤ │ WebDriver Layer │ ← Selenium WebDriver ├─────────────────────────────────────┤ │ Browser Layer │ ← Chrome Browser └─────────────────────────────────────┘

Project Structure

insider-selenium/ ├── src/test/java/ │ ├── base/ │ │ └── BaseTest.java # Base test class with common setup │ ├── objects/ # Page Object classes │ │ ├── HomePage.java # Homepage interactions │ │ ├── MorePage.java # Navigation menu │ │ ├── CareersPage.java # Careers page │ │ ├── QAPage.java # QA department page │ │ ├── OpenPositionPage.java # Job listings page │ │ └── VerifyLeverPage.java # Lever application page │ ├── runner/ │ │ └── TestRunner.java # Test suite runner │ └── test_scenarios/ │ └── VerifyQAJobs.java # Main test scenario ├── src/test/resources/ │ └── test-config.properties # Test configuration ├── drivers/ # ChromeDriver files ├── pom.xml # Maven configuration └── README.md # Project documentation

Key Components

Installation and Usage

Prerequisites

  • Java 11 or higher
  • Maven 3.6 or higher
  • Chrome Browser (latest version)
  • Git

Installation Steps

1. Clone the Repository

git clone https://github.com/erdncyz/insider-selenium.git
cd insider-selenium

2. Install Dependencies

mvn clean install

Execution Commands

Run All Tests

mvn test

Run Specific Test

mvn test -Dtest=VerifyQAJobs

Run in Headless Mode

mvn test -Dselenium.headless=true

Clean and Compile

mvn clean test-compile

Code Analysis

BaseTest.java - Base Test Class

Provides common configuration for all test classes:

  • WebDriver Initialization: ChromeDriver setup with WebDriverManager
  • Chrome Options: Optimized browser settings for performance
  • Timeout Configuration: Implicit wait (10s), Page load timeout (60s)
  • Automatic Cleanup: Driver cleanup after each test

Page Object Classes

HomePage.java

  • verifyInsiderLogo(): Verifies Insider logo is displayed
  • acceptAllPermissions(): Accepts cookie permissions
  • clickCompanyMenu(): Clicks on Company menu

CareersPage.java

  • verifyCareersPage(): Verifies career page elements
  • selectQualityAssurance(): Selects QA department with scroll and JavaScript click

OpenPositionPage.java

  • openPositionPage(): Clicks "See all QA jobs" button

Optimized Locators

  • Company Menu: //a[contains(text(),'Company')]
  • Careers Link: //a[normalize-space()='Careers']
  • See All Teams: //a[text()='See all teams']
  • Quality Assurance: //h3[normalize-space()='Quality Assurance']
  • See All QA Jobs: //a[contains(text(),'See all QA jobs')]

Best Practices Implemented

1. Page Object Model (POM)

  • Separate class for each page
  • Locators defined as private fields
  • Descriptive method names
  • Encapsulation of page interactions

2. Exception Handling

  • Try-catch blocks for robust error handling
  • InterruptedException properly managed
  • JavaScript click fallbacks for element interactions

3. Wait Strategies

  • Implicit Wait: 10 seconds
  • Page Load Timeout: 60 seconds
  • Explicit waits where needed
  • Thread.sleep for page transitions

4. Code Organization

  • Logical package structure
  • Clear separation of concerns
  • Consistent naming conventions
  • Clean and readable code

5. Element Interaction

  • JavaScript click fallbacks for intercepted elements
  • Scroll to element before interaction
  • Optimized XPath and CSS selectors
  • Robust element location strategies

Troubleshooting

Common Issues and Solutions

1. ElementClickInterceptedException

Issue: Element is covered by another element (e.g., navbar)
Solution: JavaScript click fallback implemented in CareersPage.java

2. NoSuchElementException

Issue: Element not found on page
Solution: Check locators and wait strategies

3. TimeoutException

Issue: Page load timeout
Solution: Increased page load timeout to 60 seconds

4. ChromeDriver Version Issues

Issue: ChromeDriver version mismatch
Solution: WebDriverManager handles automatic version management
