üéØ Project Overview

Project Purpose

This project is a Selenium WebDriver-based test automation project developed to automatically test QA (Quality Assurance) positions on the career page of Insider company's website.

Test Scenario

  1. Homepage Verification: Verification of Insider logo visibility and acceptance of cookie permissions
  2. Navigation: Transition from "More" menu to "Careers" page
  3. Career Page: Verification that the page loads correctly
  4. QA Department: Selection of Quality Assurance department
  5. Position List: Listing of QA positions
  6. Open Position: Display of open positions in Istanbul location
  7. Lever Page: Verification of Lever application page

üèóÔ∏è Technical Architecture

Architectural Approach

The project uses the Page Object Model (POM) design pattern. This approach:

  • Maintainability: Enables easy code changes
  • Reusability: Ensures test codes can be reused
  • Ease of Maintenance: Allows web elements to be managed from a single location

üß™ Test Scenarios

Main Test Case: QA Jobs Verification

This test case validates the complete workflow for finding and applying to QA positions at Insider.

Test Steps:

  1. Homepage Verification: Visit https://useinsider.com/ and verify the Insider home page is opened
  2. Navigation to Careers: Select "Company" menu ‚Üí "Careers" and verify the Career page with Locations, Teams, and Life at Insider blocks
  3. QA Department Access: Navigate to https://useinsider.com/careers/quality-assurance/ and click "See all QA jobs"
  4. Job Filtering: Filter jobs by Location: "Istanbul, Turkey" and Department: "Quality Assurance"
  5. Job List Verification: Verify all jobs contain "Quality Assurance" in Position and Department, and "Istanbul, Turkey" in Location
  6. Application Form: Click "View Role" button and verify redirection to Lever Application form page

Page Object Classes:

1. HomePage.java - Home Page Operations

- verifyInsiderLogo()     // Verifies Insider logo is displayed

- verifyLocationsBlock()  // Verifies Locations section
- verifyTeamsBlock()      // Verifies Teams section

- selectQualityAssurance() // Selects QA department

Optimized Locators:

  • Teams Section: //h2[contains(text(),'Teams')]

‚öôÔ∏è Installation and Usage

Test Case Requirements

  • Programming Language: Java with Selenium WebDriver (preferred)
  • Framework Restrictions: No BDD frameworks allowed (Cucumber, Quantum, Codeception, etc.)
  • Design Pattern: Must fully meet Page Object Model (POM) requirements
  • Element Locators: Optimized XPath and CSS selectors for buttons, dropdowns, and other elements
  • Assertions: Must use assertions in test cases for validation
  • Code Quality: Clean, readable, and maintainable code

System Requirements

  • Java 11 or higher
  • Maven 3.6 or higher
  • Chrome Browser (latest version)
  • Git

Installation Steps

1. Download Project

git clone https://github.com/your-username/insider-test-automation.git
cd insider-test-automation

2. Install Dependencies

mvn clean install

3. Download Chrome Driver

  • Download the appropriate version for your operating system from ChromeDriver
  • Place it in the drivers/ folder

Execution Options

Running with Maven

# Run all tests
mvn test

# Run only QA Jobs test
mvn test -Dtest=test_scenarios.VerifyQAJobs

# Run test suite
mvn test -Dtest=runner.TestRunner

# Run in headless mode
mvn test -Pheadless

Running with Maven Commands

# Run all tests
mvn clean test

# Run only QA Jobs test
mvn test -Dtest=test_scenarios.VerifyQAJobs

# Run test suite
mvn test -Dtest=runner.TestRunner

# Run in headless mode
mvn test -Pheadless

# Clean and install
mvn clean install

üéØ Best Practices

1. Page Object Model (POM)

  • Separate class created for each page
  • Locators defined as private fields
  • Method names made descriptive

2. Exception Handling

  • Try-catch blocks used
  • InterruptedException properly handled
  • Test errors are logged

3. Wait Strategies

  • Implicit Wait: 15 seconds
  • Explicit Wait: 10 seconds
  • Page Load Timeout: 30 seconds

4. Code Organization

  • Package structure logically organized
  • Utility classes in separate package
  • Test scenarios in separate package

5. Configuration Management

  • Configuration with properties file
  • Different environments with Maven profiles
  • Environment variables support
